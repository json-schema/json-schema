JSON Schema

What is JSON Schema

JSON Schema is a JSON based format for describing JSON data. JSON Schema is to JSON as XSD is to XML.

JSON Schema is a proposed Internet draft defining a JSON media type (application/schema+json) with the following goals:

Validation
You can use JSON Schema to validate your JSON data.
Documentation
You can extract documentation information from a JSON Schema, match this documentation to your data, and use that information for user interaction.
Hyperlinking
You can pair your JSON data with the defining JSON Schema to build hyperlinks into parts of that JSON data.

JSON Schema standards route

The last IETF published draft was v3 which expired on May 26, 2011 as is the nature of drafts. In response to a request by two json schema hacktivists, after creating the json-schema project on github in August 2012, Kris Zyp and Gary Court, with their blessings bestowed, sponsored the json-schema.org domain and the json-schema google group to the json-schema community, symbolizing a passing of the flame to the next torch bearers. Now committed to see json-schema specification through to official RFC acceptance once and for all.

The goal, to realize the official json schema specification to RFC acceptance.

The strategy, a first time right approach, by merging the existing community under a single banner to encourage and facilitate involvement as a combined team effort; frequent release of previews for public review and to reach majority consensus, akin to the official IETF process and; the combined specification divided into manageable and separate bite sized volumes with specific focus and context to simplify execution, consumption and acceptance driven in parallel by volunteers with particular interest and buy-in there in.

We would love to hear your thoughts/suggestions/ideas/concerns about our strategy, please start a discussion on the issue tracker and meet your fellow json-schema aficionados.

The following volumes have already received active attention. Once we are happy that you are happy then hopefully the official submission would be no more than a formality.

json-schema-core
Introduces JSON schema, states purpose, defines its IANA media type, the schema document and main keywords, nesting and relations of schemas, its primitive data types and their equality definitions. Additionally it introduces the separate document approach and sets formal guidelines.
json-schema-validation
Introduces schema validation and establishes interoperability guidelines. Specifies the validation related keywords specific to each primitive data type, includes meta information, semantics and relationships.
json-schema-hypermedia
Introduces the concept of hypermedia and defines its application on JSON data. It encompasses links, relation types, content types and includes data submission guidelines.

Each preview release is accompanied by a release notification on the issue tracker which is where you are given the opportunity to raise any concerns, point out discrepancies or similar fallacies, identify missing, incomplete or ambiguity and it cannot be stressed enough just how important your involvement is to our inevitable success or forever hold your peace, instead.

JSON Schema also uses two other specifications, both currently still under review by the IETF:

  • JSON Reference
  • JSON Pointer

Reference schemas

The following are schemas that are available for referencing (more to come):

Core Schema
The proposed core schema currently in review.
Core Meta-Schema
This is the meta-schema definition as it stood at draft 03 submission and currently in active use, for schemas themselves (the type/validation part of schemas).
Hypermedia Meta-Schema
This is the extended full meta-schema definition for schemas with link definitions (hypermedia schemas).
Hypermedia Links Meta-Schema
The schema definition which defines schemas defining links as used with hypermedia schemas.
JSON Referencing
This is the schema that defines a generic linking mechanism called JSON referencing (which is used by the meta-schema for links).
JSON Schema Interfaces
This is the meta-schema definition for schemas which define class-style method interfaces for programming languages.
Geographic Coordinate
A schema definition for geographical coordinates.
Card
A schema for microformat style representation of a person, company, organization, or place.
Calendar
A schema for microformat style representation of an event.
Address
A schema for microformat style representation of an address.
