JSON Schema based implementations

Preliminary note

Implementations below are written in different languages, and support part, or all, of the specification. These projects are not affiliated with json-schema.org nor do we make any claim to their usability for any particular purpose. This index purely serves as a record of JSON Schema adoption, purposed primarily to motivate the need for an accepted standards specification.

The implementations are classified by type of application, programming language, applicable license and version of JSON Schema in use and is sorted by application type, language and ordered by most recently updated.

Version numbers are indicative only and impractical for comparison, projects frequently updated are indications of good health and would suggest that support may be readily available.

Type legend:

The type of JSON Schema application distinguishes the usability of a library to be applicable for a specific purpose. The following descriptions serve to clarify these distinctions.
Validation (V)
The library applies a JSON schema definition to JSON data and reports on any constraint violations that may be present.
Documentation (D)
The library facilitates the creation of human readable documentation based on the definitions contained in a JSON Schema document.
Generation (G)
The library facilitates the generation or creation of JSON Schema documents in some way.

With your help we can make this the de facto list of JSON Schema based solutions available. Please leave notice on the issue tracker of any additional projects you may come across not yet listed. Thank you for your continued participation.

Please note: Lack of recent activity alone should not immediately disqualify a project unless it bears no evidence of maturity. Popularity can be judged by the number of watchers or forks whereas the number of open issues may reflect on both the quality and the amount of attention you can expect to receive from the team when issues arise but again it is only an assumption at best. More popular projects will generate more issues and every project make different use of their issue trackers, the open issues might be discussions for example which will likely remain open and shows that the project owners are considerate to user input which is always a good thing.

We only aim to do the initial legwork on your behalf by collecting the comparable indicators, we strongly recommend that further investigation are done before settling on any specific solution.

The unity list
